# Конфетная фабрика (веб-версия)

Онлайн-монитор командного турнира по решению задач с автоматическим учётом
решений через [informatics.msk.ru](https://informatics.msk.ru). Реализация по
ТЗ `TZ-candy-factory-web.pdf`.

Один статический бинарник на Go: HTTP-сервер (серверный рендеринг + JSON-API
автообновления), фоновый опросчик информатикса, SQLite-хранилище. Статика и
шаблоны встроены в бинарник.

## Возможности

- **Админка** `/admin`: создание/настройка/запуск/завершение игр, живое табло
  с покупками задач и тестов, журнал событий (добавление/редактирование/
  удаление/отключение), предупреждения валидации, аномалии, пароли команд.
- **Несколько игр параллельно и независимо.**
- **Публичное табло** `/g/{id}`: запасы, скорость, сетка задач с цветами
  состояний, гистограмма, таймер — без ссылок на задачи (в том числе в JSON).
- **Страница команды** `/g/{id}/team` (логин/пароль): то же табло плюс своя
  таблица задач; порядок задач внутри строки перемешан индивидуально для
  каждой команды. Команда **сама покупает задачи** кнопкой «Купить» (при
  нехватке средств или производительности покупка блокируется); **ссылка на
  условие появляется только после покупки** — до этого задача никак не
  идентифицируется. Покупки тестов по-прежнему вводит администратор.
- **Автозачёт**: решения подтягиваются с информатикса по аккаунтам команд;
  посылки, которые нельзя применить автоматически, попадают в аномалии
  («Принять»/«Отклонить»).
- **Журнал событий — единственный источник истины**: всё состояние
  детерминированно пересчитывается из журнала; редактирование прошлого
  корректно меняет цифры на всех страницах.

## Сборка

Требуется Go 1.22+ (CGo не нужен — драйвер SQLite modernc.org/sqlite):

```sh
go build -o candy-factory .
# или под сервер:
GOOS=linux GOARCH=amd64 CGO_ENABLED=0 go build -o candy-factory .
```

Тесты:

```sh
go test ./...
```

## Быстрый старт локально

```sh
./candy-factory -listen :8080 -data-dir ./data
```

При первом запуске создаётся каталог данных и
`data/credentials/admin_credentials.json` с логином `admin` / паролем
`change_me` — **поменяйте пароль**. Файл читается при каждой попытке входа,
рестарт не нужен.

Для автозачёта заполните `data/credentials/informatics_credentials.json`
(пример — в `data.example/`):

```json
{
  "username": "сервисный_логин",
  "password": "пароль",
  "base_url": "https://informatics.msk.ru"
}
```

`base_url` опционален; можно указать зеркало `https://informatics.mccme.ru` —
переключается одним местом. Без корректного файла веб-сервер работает, но
опрос информатикса не идёт (в админке — баннер с ошибкой).

## Параметры запуска

| Флаг             | По умолчанию | Смысл                                        |
|------------------|--------------|----------------------------------------------|
| `-listen`        | `:8080`      | адрес HTTP-сервера                           |
| `-data-dir`      | `./data`     | каталог данных                               |
| `-poll-interval` | `30s`        | период опроса информатикса                   |
| `-page-refresh`  | `1s`         | период автообновления страниц (отдаётся клиенту) |

Переменные окружения `CANDY_LISTEN` и `CANDY_DATA_DIR` перекрывают дефолты,
если флаг не задан явно.

### Каталог данных

```
data/
  app.db                          # SQLite
  secret                          # ключ подписи cookie (генерируется сам)
  credentials/
    admin_credentials.json        # {"login": "...", "password": "..."}
    informatics_credentials.json  # сервисный аккаунт информатикса
  cache/
    informatics_state.json        # кеш опросчика (max_run_id по аккаунтам)
  server.log                      # текстовый лог приложения
```

Все времена хранятся и передаются в UTC; отображение на страницах — в часовом
поясе сервера (задаётся переменной `TZ`, для турниров в Москве —
`TZ=Europe/Moscow`, см. unit-файл).

При рестарте (в том числе после `kill -9`) состояние полностью
восстанавливается из БД и кеша: идущие игры продолжаются без потерь, дублей
решений не возникает (дедупликация по `run_id` в БД).

## Развёртывание на сервере (Ubuntu/Debian)

```sh
# 1. Пользователь и каталог
sudo useradd -r -s /usr/sbin/nologin candy
sudo mkdir -p /opt/candy-factory
sudo cp candy-factory /opt/candy-factory/
sudo cp -r data.example /opt/candy-factory/data
sudo chown -R candy:candy /opt/candy-factory

# 2. Настройте креды
sudo -u candy editor /opt/candy-factory/data/credentials/admin_credentials.json
sudo -u candy editor /opt/candy-factory/data/credentials/informatics_credentials.json

# 3. systemd (Restart=always уже в unit-файле)
sudo cp deploy/candy-factory.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now candy-factory
systemctl status candy-factory

# 4. Логи
journalctl -u candy-factory -f
tail -f /opt/candy-factory/data/server.log
```

HTTPS — забота reverse-proxy перед приложением; пример конфига nginx —
`deploy/nginx.conf.example`:

```sh
sudo cp deploy/nginx.conf.example /etc/nginx/sites-available/candy-factory
# поправьте server_name и пути к сертификатам
sudo ln -s /etc/nginx/sites-available/candy-factory /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
```

### HTTPS при доступе по IP (без домена)

Обычный сертификат Let's Encrypt на голый IP не выпустить — используется
самоподписанный (браузер один раз покажет предупреждение, участники нажимают
«Дополнительно → Перейти на сайт»). Готовый конфиг —
`deploy/nginx-ip-selfsigned.conf`:

```sh
IP=203.0.113.10   # ваш внешний IP
sudo openssl req -x509 -nodes -newkey rsa:2048 -days 825 \
  -keyout /etc/ssl/private/candy-selfsigned.key \
  -out    /etc/ssl/certs/candy-selfsigned.crt \
  -subj "/CN=$IP" -addext "subjectAltName=IP:$IP"

sudo cp deploy/nginx-ip-selfsigned.conf /etc/nginx/sites-available/candy-factory
sudo rm -f /etc/nginx/sites-enabled/default
sudo ln -sf /etc/nginx/sites-available/candy-factory /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
```

## Как провести турнир

1. Зайдите в `/admin`, «Создать игру»: название, число уровней `n` (2–6),
   параметры уровней (предзаполнены дефолтами), стартовые запасы/скорость,
   длительность, ровно `n` ссылок на задачи информатикса на каждый уровень
   (`https://informatics.msk.ru/mod/statements/view.php?chapterid=...`,
   зеркала и `#1` на конце принимаются), команды (минимум 2: имя,
   `informatics user_id` из URL профиля `/user/profile.php?id=...`, логин и
   пароль страницы команды).
2. Раздайте командам ссылки и пароли (секция «Команды» на странице игры).
3. «Старт сейчас» (или задайте плановое время старта заранее — игра стартует
   автоматически). После старта состав задач и параметры не редактируются.
4. Задачи команды покупают сами со своей страницы; админ может вводить любые
   покупки (задач и тестов) кликом по ячейке живого табло — при нехватке
   средств админка предупредит и попросит подтверждение (команде такая
   покупка блокируется). Кнопка «Продлить (мин)» на странице игры принимает
   любое число минут, в том числе отрицательное (сокращение).
5. Решения подтягиваются автоматически (период `-poll-interval`); опрос
   продолжается 15 минут после конца игры, чтобы зачесть посылки «на флажке».
   Посылки, которые нельзя применить (задача не куплена, вне времени), падают
   в «Аномалии» — решите их кнопками «Принять»/«Отклонить».
6. Ошибочные действия исправляются редактированием журнала событий: состояние
   пересчитывается мгновенно на всех страницах.
7. По окончании — «В архив»: игра исчезает с главной страницы.

## Маршруты

| Маршрут                              | Доступ         | Содержимое                        |
|--------------------------------------|----------------|-----------------------------------|
| `GET /`                              | все            | список неархивных игр             |
| `GET /g/{gameId}`                    | все            | публичное табло                   |
| `GET /g/{gameId}/team`               | форма входа    | логин команды                     |
| `GET /g/{gameId}/team/{teamId}`      | команда/админ  | страница команды                  |
| `GET /api/g/{gameId}/state`          | все            | JSON состояния (без ссылок)       |
| `GET /api/g/{gameId}/team/{id}/state`| команда/админ  | JSON + ссылки и порядок команды   |
| `GET,POST /admin…`                   | админ          | админка                           |
